Git

Created2025-03-21|Updated2025-04-11
|Post Views:

以下是Git的基本使用方法：

1. 创建/克隆仓库

  • 创建新仓库
    • 在本地项目文件夹中初始化仓库：git init
    • 将项目与远程仓库关联：git remote add origin 远程仓库URL
  • 克隆远程仓库
    • 克隆远程仓库到本地：git clone 远程仓库URL

2. 添加文件到暂存区

  • 添加单个文件：git add 文件名
  • 添加所有更改的文件：git add .

3. 提交更改到本地仓库

  • 提交更改并附上提交信息：git commit -m "提交信息"

4. 查看仓库状态

  • 查看当前仓库状态（包括已修改但未暂存的文件等）：git status
  • 查看提交历史：git log

5. 推送更改到远程仓库

  • 推送本地提交到远程仓库的指定分支：git push origin 分支名

6. 分支管理

  • 查看所有分支：git branch
  • 创建新分支：git branch 分支名
  • 切换分支：git checkout 分支名
  • 合并分支：git merge 要合并的分支名 到当前分支

7. 拉取远程仓库的更新

  • 获取远程仓库的更改并合并到当前分支：git pull origin 分支名

8. 比较不同版本

  • 比较当前工作目录与上次提交的差异：git diff
  • 比较两个特定提交之间的差异：git diff 提交ID1 提交ID2

9. 标签管理

  • 查看所有标签：git tag
  • 创建新标签：git tag 标签名
  • 推送标签到远程仓库：git push origin 标签名

10. 解决冲突

  • 当合并或拉取时出现冲突，需要手动编辑冲突的文件，解决冲突后，添加文件并提交。

11. 撤销操作

  • 撤销工作目录中单个文件的更改：git checkout 文件名
  • 撤销上次提交：git reset --soft HEAD^（保留更改到暂存区）或git reset --hard HEAD^（丢弃更改）

12. 配置Git

  • 查看当前配置：git config --list
  • 设置用户信息：git config --global user.name "用户名"和git config --global user.email "邮箱地址"

以上是Git的一些基本使用方法，涵盖了日常使用中的常见操作。通过合理运用这些命令，可以有效地进行版本控制和团队协作。
